  • The customer segmentating for sales data uses RFM (Recency, Frequency, Monetary) analysis. Therefore, anyone who wants to learn more about RFM analysis can visit this website
  • The clustering method used for this app is K-Means clustering method. For more information, visit this website

How to use this application

This application is intended for segmenting customer data extracted from FreshSales and FreshDesk. However, any sales and tickets data will work if there is enough information.

The required fields include:

  • For sales RFM segmenting:
    • AccountID: The ID of the account who did the purchase
    • CloseDate: The date that the deal was closed on
    • DealValue: The value of deals (in a consistent currency)
    • DealStage: The status of the deal (Won for deals won, and Lost for deals lost, and others)
  • For tickets segmenting:
    • Group Company: The group that the client's company belongs to
    • Brand: The brand of the client's company
    • Client code: The client code provided by FreshDesk
    • Customer: Whether or not a customer raised that ticket (Yes or No)
    • Closed time: The time that the ticket was closed
WARNING: all of the columns above need to be exactly named like that, or else the system won't work.

What is this app doing?

This app is for customer segmentation, a way to divide customers into groups to strategize approaches to different customer clusters.

By using the data provided, this app uses relevant fields to divide the customer base.

Note: when divided, the clusters will not the ranked, e.g. cluster 1 is not 'better' or 'worse' than cluster 2 customers. You will also have the option to download the data with every companies clustered, which can be used for your own analysis.
